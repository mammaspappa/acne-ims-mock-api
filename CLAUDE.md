## Project

Acne Studios IMS (Inventory Management System) — a mock API server for a hackathon simulating the full supply chain of a luxury fashion company. The system models 80+ retail stores, 20+ e-commerce sites, and ~800 wholesale partners across 50+ countries.

## Tech Stack

- **Runtime:** Node.js + TypeScript
- **Framework:** Fastify 5.x (REST API)
- **Data layer:** In-memory singleton store (no database) with JSON seed data + snapshot/reset
- **Auth:** JWT (HS256, `@fastify/jwt`)
- **Fake data:** `@faker-js/faker`
- **Build/Run:** `tsx` (no separate build step needed)
- **Testing:** Vitest

## Common Commands

```bash
cd acne-mock-api
npx tsx src/index.ts          # Start server on port 3000
npx tsc --noEmit              # Type-check without emitting
npx vitest                    # Run tests
```

    reports/                  # Analytics dashboards, KPIs

## Key Conventions

- **All data is in-memory.** The `Store` singleton (`store/Store.ts`) holds all collections. `POST /admin/reset` restores from snapshot.
- **Passphrase for simulation:** `acne-hackathon-simulate-2026` (required for start/stop/scenario activation).
- **All test user passwords:** `hackathon2026`.
- **ID generation:** `nanoid` via `utils/id.ts`.
- **Time travel:** `utils/date.ts` exports `now()` which respects `setMockNow()` for simulated time.
- **State machines:** PO and SO status transitions are enforced in `po.state-machine.ts` and `so.state-machine.ts`.
- **Fastify route typing:** Handlers use `as any` at route registration to work around Fastify's schema-to-type inference gap. This is intentional.
- **Simulation events:** Tagged with `[SCENARIO]` prefix and `_scenario` details field when generated by an active scenario.

## Simulation & Scenarios

The simulation (`modules/admin/simulation.ts`) generates realistic mock events across all external systems. It supports:
- Speed multiplier (1x to 1000x)
- 25 weighted event generators (commerce, fulfillment, RFID, payments, supply chain, AI, etc.)
- **16 activatable scenarios** (`modules/admin/scenarios.ts`) that inject disruptive events:
  - **Demand:** VIRAL_PRODUCT, CELEBRITY_ENDORSEMENT, FLASH_SALE_GONE_WRONG
  - **Market:** GEOPOLITICAL_DISRUPTION, MARKET_RECESSION, GLOBAL_ECONOMIC_SLOWDOWN, CURRENCY_CRISIS
  - **Supply:** SUPPLIER_DISRUPTION, LOGISTICS_BOTTLENECK, RAW_MATERIAL_SHORTAGE
  - **Quality:** QUALITY_CRISIS, COUNTERFEIT_SURGE
  - **Operational:** WAREHOUSE_OUTAGE, PAYMENT_PROVIDER_OUTAGE, CYBER_INCIDENT
  - **External:** WEATHER_ANOMALY
- Scenarios can be manually activated or auto-triggered at random intervals (`autoScenarios: true` on simulation start).
